The core component for ensuring your connect wallet integration works as expected. This component must wrap your application code and be nested within a `WagmiConfig`.

## Usage

```typescript
import {ConnectWalletProvider} from '@shopify/connect-wallet';

export const YourApp = ({chains, children, client}) => {
  return (
    <WagmiConfig client={client}>
        // highlight-next-line
        <ConnectWalletProvider chains={chains}>
            {children}
        // highlight-next-line
        </ConnectWalletProvider>
    </WagmiConfig>
  );
}
```

:::note

Ensure that ConnectWalletProvider is wrapped within the [required `WagmiConfig`](../getting-started#providers).

:::


## Props

### chains
| | |
| - | - |
| **Description**       | A list of the chains supported by your application. |
| **Type**              | [`Chain[]`](https://wagmi.sh/react/providers/configuring-chains) |
| **Required**          | `true` |
| **Default value**     | `undefined` |

### signOnConnect
| | |
| - | - |
| **Description**       | A boolean value indicating whether you want to request signature as soon as a wallet connects or not. |
| **Type**              | `boolean` |
| **Required**          | `false` |
| **Default value**     | `true` |

### wallet
| | |
| - | - |
| **Description**       | If you are managing any wallet connections outside of the application and want to inject a connected wallet, you can do so by passing a wallet value here. |
| **Type**              | [`Wallet`](../types/wallet) |
| **Required**          | `false` |
| **Default value**     | `undefined` |
